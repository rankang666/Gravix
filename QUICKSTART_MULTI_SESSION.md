# 多会话功能快速启动指南

## 🚀 快速开始

### 1. 重启服务器

```bash
# 停止现有服务器
pkill -f "python.*run_all.py"

# 启动新服务器
python3 run_all.py
```

### 2. 打开浏览器

访问：**http://localhost:8765**

### 3. 界面说明


## 💡 基本操作

### 创建新会话
1. 点击左上角 **"+ 新建"** 按钮
2. 自动创建并切换到新会话
3. 开始新的对话

### 切换会话
- 点击左侧列表中的任意会话
- 自动加载该会话的历史消息
- 可以在不同会话间自由切换

### 删除会话
1. 鼠标悬停在要删除的会话上
2. 点击右上角的 **"×"** 按钮
3. 确认后删除会话

## 📱 移动端使用

在手机上使用时：
- 点击顶部 **"☰"** 按钮展开/收起会话列表
- 侧边栏会滑出，方便操作
- 再次点击收起侧边栏

## ⚙️ 高级功能

### 会话数据保存
所有会话自动保存在 `data/sessions.json`
- 每次操作自动保存
- 重启服务器数据不丢失
- 可以手动备份此文件

### API 调用示例

```bash
# 获取所有会话
curl http://localhost:8765/api/sessions

# 创建新会话
curl -X POST http://localhost:8765/api/sessions \
  -H "Content-Type: application/json" \
  -d '{"title": "我的新会话"}'

# 切换会话
curl -X PATCH http://localhost:8765/api/sessions/{session_id}

# 删除会话
curl -X DELETE http://localhost:8765/api/sessions/{session_id}
```

## 🔧 故障排查

### 问题：看不到会话列表
**解决**：
1. 刷新浏览器页面
2. 检查浏览器控制台是否有错误
3. 确认服务器已启动

### 问题：无法创建新会话
**解决**：
1. 检查网络连接
2. 查看浏览器控制台错误信息
3. 确认 API 端点可访问

### 问题：会话数据丢失
**解决**：
1. 检查 `data/sessions.json` 文件是否存在
2. 检查文件权限
3. 从备份恢复（如有）

## 📊 数据管理

### 备份会话数据
```bash
# 备份
cp data/sessions.json data/sessions.json.backup

# 恢复
cp data/sessions.json.backup data/sessions.json
```

### 清空所有会话
```bash
# 删除会话文件
rm data/sessions.json

# 重启服务器会自动创建默认会话
```

### 导出单个会话
```bash
# 获取会话数据
curl http://localhost:8765/api/sessions/{session_id} > session_export.json
```
